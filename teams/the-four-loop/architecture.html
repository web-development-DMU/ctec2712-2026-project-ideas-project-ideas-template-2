Architecture Documentation

High-level architecture for SourceFlow (Personal Shopping Request System).

System overview

  • Client: HTML/CSS pages (documentation + UI screens)
  • Server: JavaScript server handling routes + validation
  • Database: SQLite schema (customers, requests, statuses, request_notes)
Browser (UI pages)

   v

   |

   v
SQLite Database

Data flow

  1. Customer submits a request (item, size, budget, preferences).
  2. Server validates and inserts into requests with default status New.
  3. Concierge updates status (In Progress → Sourced → Completed).
  4. Concierge adds notes into request_notes.
  5. Customer views status and updates via request list/detail.

Non-functional considerations

  • Accessibility: semantic HTML, labels, contrast, keyboard-friendly nav.
  • Maintainability: clean schema + predictable routes.
  • Security (future): auth for concierge/admin actions.
