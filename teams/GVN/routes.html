Navigation URLs

GET/index/login

    Once the user presses the submit button, the server runs an SQL query that checks only the records which match the login details entered by the user. If the server finds a matching record, it sends a successful login response (200 OK) and redirect user to the main page (303). If no match is found, the server returns an unsuccessful response, and the user is shown an error message.

index/register - Will display the form UI to fill out the information

POST/index/register/createUser - Create a new user account by sending registration data to the server

    Once the user presses the submit button, the server recieves the data through POST request, the server will validate the data, the password will be hashed, the server runs an SQL query that will create a new record in the database, the server will return a successful response (200 OK) if the data is added or unsuccessful (400) if the data already exists in the database (eg email)

index/Exercise

index/Exercise/viewExercise - Will display all the exercises stored in the system by retrieving them from the database and showing them to the user

    The server will use GET method to request, the server will retrieve all the records from the exercise database using SQL query, the server will send back the data to the user, the exercise page will display all the exercise records

GET/index/myRoutines/userRoutines - List all of the user created/saved routines

    The server will identify the logged in user, the server will retrieve users created/saved routines from the database using SQL query, the server will send back the database back to the user, the page will display all the routines with the information and will have feature to update/remove.

GET/index/myRoutine/userRoutines/updateRoutine/:routineid - Page to update a selected routine. add/remove an exercise from the routine.

    The server will identify the routine using the routineid from the URL, retrieve the data for the routine from the database using SQL query, the page will display all the data for the routine including the routine name and the exercises, the user can modify the routine, the user will have option to save the modified routine. Once the user submits the data, the server will take all the data using POST method and update the database record for the routine using SQL query.

Admin Page Routes/URLs

-- User --

GET/admin/user - List all of the users/members in the admin dashboard

    The server will identify the user to authenicate if the user is admin, if not then the user will get an error message 404 and will not be allowed on the page. The server will recieve a request to display all the users into the user management page using SQL query, it will retrieve all the user records from the database, the server will render all the user records into the user management page. The admin will have option to update/remove users from the database.

POST/admin/user/createUser - It will be used by the administrator to create a new user account in the database. It uses the POST method because it is sending new data to the server that will be inserted into the database.

    The server will retrieve entered data using the POST request, the server will validate all the data, Once validated the server will create a new record for the user in the database using SQL query, send successful response (200 OK) if added and redirect to the user management page returning (303) status code, if not unsuccessful (400) response with error message (eg record already existing by comparing all the emails in the database)

GET/admin/user/updateUser/:userid - It will be used by the administrator for updating a specific user's account details

    The server will identify the user using the userid from the URL, retrieve the data for the user from the database using SQL query, the page will display all the data for the user, the admin will have option to modify the data and save. Once the admin submits the data, the server will take all the data using POST method and update the database record for the user using SQL query.
