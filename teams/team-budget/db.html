Database Structure

This page documents the database design for Budget Buddy (Budget Tracker). The application uses a relational database with tables linked by foreign keys.

Tables

1) users

  • user_id (PK)
  • name
  • email (unique)
  • password_hash

2) categories

  • category_id (PK)
  • name (e.g., Food, Rent, Transport)

3) transactions

  • transaction_id (PK)
  • user_id (FK → users.user_id)
  • category_id (FK → categories.category_id)
  • type (Income / Expense)
  • amount
  • date
  • description

4) budgets

  • budget_id (PK)
  • user_id (FK → users.user_id)
  • category_id (FK → categories.category_id)
  • month (e.g., 2026-02)
  • monthly_limit

Relationships

  • One user can have many transactions.
  • One category can be used by many transactions.
  • One user can set many budgets (usually one per category per month).
  • Each budget belongs to one user and one category.

Notes

  • Foreign keys help connect data and enforce consistency.
  • Budgets are compared against total expenses in that category for the chosen month.
