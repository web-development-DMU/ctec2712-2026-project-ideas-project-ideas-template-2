Database Structure

Budget Buddy uses a relational database with tables linked using foreign keys to enforce consistency. Tables are designed to support CRUD operations for transactions and budgets.

Tables

1) users

  • user_id (Primary Key)
  • name
  • email (unique)
  • password_hash

2) categories

  • category_id (Primary Key)
  • name (e.g., Food, Rent, Transport)

3) transactions

  • transaction_id (Primary Key)
  • user_id (Foreign Key → users.user_id)
  • category_id (Foreign Key → categories.category_id)
  • type (Income / Expense)
  • amount
  • date
  • description

4) budgets

  • budget_id (Primary Key)
  • user_id (Foreign Key → users.user_id)
  • category_id (Foreign Key → categories.category_id)
  • month (e.g., 2026-02)
  • monthly_limit

Relationships

  • One user can have many transactions.
  • One category can be used by many transactions.
  • One user can set many budgets (usually one per category per month).
  • Each budget belongs to one user and one category.

Notes

  • Foreign keys help connect data and enforce consistency.
  • Budgets are compared against total expenses in a category for a selected month.
