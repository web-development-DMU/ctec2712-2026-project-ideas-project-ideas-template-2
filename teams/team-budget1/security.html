Security Considerations

SQL Injection Prevention

User input will never be inserted directly into SQL queries. Prepared statements and parameterised queries will be used to prevent SQL injection attacks.

Cross-Site Scripting (XSS) Prevention

All user input displayed on the page will be properly escaped to prevent malicious scripts from executing in the browser.

Input Sanitisation

User data will be validated and cleaned before being stored in the database. Invalid or suspicious input will be rejected.

Server-Side Validation

Validation will be performed on the server to ensure users cannot bypass front-end validation.

  • Transaction name must not be empty
  • Amount must be a positive number
  • Category must be selected
  • Text inputs must be at least 2 characters long

Example Validation Logic

IF transaction_name is empty → show error
IF amount <= 0 → reject request
IF invalid data → return error message

Flash Messages

After submitting a form, the system will display temporary messages:

  • Success message when transaction is added
  • Error message when validation fails
  • Messages disappear after being shown once

Responsive Design

The application will be responsive and usable on different screen sizes.

  • Flexible layout using CSS
  • Content adapts to mobile and desktop screens
  • Readable fonts and scalable elements
