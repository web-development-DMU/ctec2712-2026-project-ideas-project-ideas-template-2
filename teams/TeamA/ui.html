User interface plan

Main pages

  • /cars — list of all cars with links to each one
  • /cars/:id — details of one car and its full service history
  • /services/new — form to log a new service record
  • /admin/cars — admin page to add, edit or delete cars
  • /admin/service-types — admin page to manage service types

User journey

A user lands on the home page and clicks through to the cars list. They select a car to see its full service history. From there they can log a new service using a simple form. Admin users can access a separate admin section to manage cars and service types.

Key forms

  • Add a car — make, model, year, registration number, owner
  • Log a service — car, service type, date, mileage, cost, notes
  • Edit a service record — same fields, pre-filled
