Description

Nutrti-Track is the name of our application and is a browser-based application that allows users to calculate nutritional content of meals by selecting foods, entering quantities and viewing totals for calories, macronutrients, and key vitamins/minerals. Built using html,css and javascript as it aims to be fast and accessible for everyday users.

NutriTrack is designed to help users make informed dietary choices without needing a complex app or login system. The application will allow users to search for foods, add them to a meal, and instantly see nutritional totals. It can pull data from a built in database. The interface will focus on clarity: a search bar, a food list, a meal builder, and a results panel. Users can adjust quantities, remove items, and save meals locally using browser storage. Administrators may have access to a backend interface to update food data, add new items, or correct nutritional values. Administrator will have access to manage the users data such as username, email and password which will be encrypted.

User Roles

  • Regular User: Searches for foods, adds food to meal, adjust quantities, view nutritional totals, saves meals locally and exports results.
  • Administrator: manages food items (add/delete), edit nutritional values, remove outdated/incorrect items, manage categories and upload bulk data.

User Requirements

  • As a user, I want to search for foods so that I can quickly find what I ate.
  • As a user, I want to add foods to a meal so that I can calculate total nutrition.
  • As a user, I want to change quantities so that the results match what I consumed.
  • As a user, I want to see calories, macros, and key nutrients so that I can track my diet.
  • As a user, I want to save meals so that I can revisit them later.
  • As an admin, I want to add new foods so the database stays up to date.
  • As an admin, I want to edit food data so that errors can be corrected.
  • As an admin, I want to upload bulk data so I can update the database efficiently.

Challenges / Risks

  • Implementing the many-to-many relationship between programmes and exercises correctly.
  • Designing a simple and intuitive user interface without relying heavily on JavaScript.
  • Managing user authentication securely.
  • Ensuring data consistency when editing or deleting exercises from programmes.

Data Requirements

  • User account information (name, email, encrypted password).
  • Workout programme details created by each user.
  • Exercise information including name and category.
  • Programme-specific data such as sets, repetitions, and exercise order.
  • Basic security measures to protect user login data.

Database Relationships

One gym member (user) can create many workout programmes, but each programme belongs to one user. This is a one-to-many relationship between Users and Programmes.

A programme can contain many exercises, and the same exercise can appear in many programmes. This creates a many-to-many relationship between Programmes and Exercises, which is resolved using the Programme_Exercises junction table.

Draft Database Structure

Users

  • user_id (Primary key)
  • name
  • email
  • password

Programmes

  • programme_id (Primary key)
  • user_id (Foreign key)
  • programme_name

Exercises

  • exercise_id (Primary key)
  • exercise_name
  • category

Programme_Exercises

This is a junction table used to link Programmes and Exercises. It stores which exercises are included in each programme, as well as programme-specific details such as sets and reps.

  • programme_id (Foreign key)
  • exercise_id (Foreign key)
  • sets
  • reps
  • exercise_order
