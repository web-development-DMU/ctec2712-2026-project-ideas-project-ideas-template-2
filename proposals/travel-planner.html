Description

So the idea is that it's an app designed for travel enthusiasts and trip planners. It allows users to create and manage their own travel itineraries. Guests/Visitors can register for an account and build personalised travel plans by selecting destinations, activities, and accommodations from a predefined travel library. Users can organise locations and activities into structured daily schedules and update details such as dates, times, and notes when needed. The system stores user accounts, destinations, activities, and travel itineraries in a database.

It needs an admin system so administrators can manage destination records and categories. Administrators can create, update, and delete destinations, activities, and travel packages to ensure the library remains accurate, relevant, and well organised. Administrators will also have access to manage user data such as username, email, and password, with all sensitive information securely encrypted.

This application will be intended for travellers who want structured trip planning, as it will help maintain organisation and efficiency, replacing scattered notes and last-minute planning. It keeps travel plans organised, accessible, and easy to modify whenever needed.

User Roles

  • Guest (Visitor): can browse public exercises/programmes but cannot create or save workout plans
  • Gym Member: creates programmes,share programmes logs workouts.
  • Administrator: manages exercises, categories, routines, users

Core Functionalities

  • Browse exercise library (available to guests and members)
  • Register and login
  • Create workout programmes
  • browser through the other programmes
  • Add exercises to programmes
  • Edit sets and repetitions
  • Delete exercises from a programme

User Requirements

  • As a guest/vistor, I want to register so that I can save my programmes.
  • As a gym member, I want to create a workout programme so that I can organise my training.
  • As a gym member, I want to add an exercise to a programme with sets and reps so that I can follow a structured routine.
  • As an administrator, I want to create exercises so that users have access to structured exercise data.
  • As an administrator, I want to manage exercise categories so that the exercise library stays organised.
  • As an administrator, i want to manage the users

Challenges / Risks

  • Implementing the many-to-many relationship between programmes and exercises correctly.
  • Designing a simple and intuitive user interface without relying heavily on JavaScript.
  • Managing user authentication securely.
  • Ensuring data consistency when editing or deleting exercises from programmes.

Data Requirements

  • User account information (name, email, encrypted password).
  • Workout programme details created by each user.
  • Exercise information including name and category.
  • Programme-specific data such as sets, repetitions, and exercise order.
  • Basic security measures to protect user login data.

Database Relationships

One gym member (user) can create many workout programmes, but each programme belongs to one user. This is a one-to-many relationship between Users and Programmes.

A programme can contain many exercises, and the same exercise can appear in many programmes. This creates a many-to-many relationship between Programmes and Exercises, which is resolved using the Programme_Exercises junction table.

Draft Database Structure

Users

  • user_id (Primary key)
  • name
  • email
  • password

Programmes

  • programme_id (Primary key)
  • user_id (Foreign key)
  • programme_name

Exercises

  • exercise_id (Primary key)
  • exercise_name
  • category

Programme_Exercises

This is a junction table used to link Programmes and Exercises. It stores which exercises are included in each programme, as well as programme-specific details such as sets and reps.

  • programme_id (Foreign key)
  • exercise_id (Foreign key)
  • sets
  • reps
  • exercise_order
