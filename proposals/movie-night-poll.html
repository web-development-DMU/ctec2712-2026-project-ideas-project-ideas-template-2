Movie Night Poll

Proposed by: Temmy Kanishk Abdishakur

Brief Description

CineVote is a web-based application that allows users to create movie night sessions, suggest films, and vote on what to watch. The application simplifies group decision-making by providing a structured voting system and displaying the winning movie.

User Requirements

  • As a user, I want to create an account so that I can join movie sessions.
  • As a user, I want to create a movie night session.
  • As a user, I want to add movie suggestions.
  • As a user, I want to vote for a movie.
  • As a user, I want to see which movie has the most votes.
  • As a user, I want to view past movie sessions.

What can the users do in the application?

  • Register and log into the system
  • Create a new movie night session
  • Join an existing session using a session code
  • Add movie suggestions to a session
  • View all suggested movies
  • Vote for one movie per session
  • See real-time vote counts
  • View the winning movie when voting closes
  • View a history of past sessions
  • The system supports full CRUD functionality:
    • Create sessions and movies
    • Read/view sessions and results
    • Update movie suggestions (optional)
    • Delete sessions or suggestions (if host)

Why do users use the application?

  • Avoid long and frustrating debates about what movies to watch
  • Make group decisions more fair and democratic
  • Save time when organising movie nights
  • Create a structured and organised voting process
  • Keep a record/log of past movie nights
  • Individual programme page
  • This application simplifies group-decision making and improves the overall movie night experience

    • Main roles within the application:

  • Registered Users
    • Create Accounts
    • Join movie sessions
    • Suggest Films
    • Vote on movies
  • Session Creator (Host)
    • Creates a movie night session
    • Shares the session code with others
    • Can close voting
    • Views final results
  • All users must be registered to participate in voting

    • Data Requirements

  • The application will store the following types of data:
  • User Data
    • Username
    • Email address
    • Encrypted password
    • Account creation date
  • Session Data
    • Session Name
    • Unique Session Code
    • Creator ID
    • Creation Date
    • Active/inactive status
  • Movie Data
    • Movie Title
    • Genre
    • Session ID
    • User who added the movie
  • Vote Data
    • User ID
    • Movie ID
    • Session ID
    • Timestamp of vote
  • Optional Comment Data
    • User ID
    • Session ID
    • Message Content
    • Timestamp

    Proposed Database Structure

  • Users
    • user_id (PK)
    • username
    • email
    • password_hash
    • created_at
  • Sessions
    • session_id (PK)
    • session_name
    • created_by (FK -> users)
    • session_code
    • created_at
    • is_active
  • Movies
    • movie_id (PK)
    • session_id (FK)
    • title
    • genre
    • added_by (FK -> users)
    • created_at
  • Votes
    • vote_id (PK)
    • user_id (FK)
    • movie_id (FK)
    • session_id (FK)
  • Comments (optional)
    • comments_id (PK)
    • user_id (FK)
    • session_id (FK)
    • message
    • created_at
